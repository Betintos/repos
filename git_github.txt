Git - распределенная система контроля версий. Предназначен для 
отслеживания ведения истории изменения файлов в вашем проекте.

Github - это сайт, куда мы можем отправлять файлы с нашего ноутбука,
чтобы другие могли посмотреть на мой проект и скачать себе

1. git init - чтобы создать скрытую директорию (.git), инициализация
2. git add . - просматривает изменения в нашем проекте и добавляет их
в промежуточное пространство (index)
3. git commit -m 'message' - фиксирует изменения из индекса в локальный репозиторий
4. git remote add {название связки (origin)} - связывает локальный и удаленный репозиторий
5. git push {название связки (origin)} main либо master - отправляет данные из локального
на удаленный репозиторий
6. git pull origin master (or main) - загружает данные из удаленного в локальный репозиторий

Закидываем изменения в готовый репозиторий
1. git add git_github.txt (название файла)
2. git commit -m 'updated file git_github.txt'
3. git push origin master или main

Стягиваем изменения из удаленного репозитория
1. git clone {ссылка удаленного репозитория}
2. git pull origin master

branch - ветка (версия проекта), она нужна для того, чтобы разрабатывать свой функционал
отдельно от других разработчиков
локальная и удаленная ветки могут быть связаны (git pull / git push)

git branch - список веток
git branch {name} - создание ветки name
git checkout {branch} - переключение между ветками
git merge {branch1} {branch2} - соединяет ветки
git mege {branch} - соединяет текущюю ветку с branch
git log - история комитов
git checkout -b {new_branch} {commit_key}

# Зафиксированное(committed) - файл уже сохранен в вашей локальной бд
# Измененное (modified)- файлы которые поменялись но не зафиксированны
# Подготовленное (staged) - измененные файлы отмеченные для включения в след коммит
